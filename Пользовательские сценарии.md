## Пользовательские сценарии

### Сценарий 1: Оператор предприятия

**Цель:** контроль текущего состояния водной среды.

1. Оператор открывает интерфейс системы на пульте.
2. Видит графики и текущие показатели по всем датчикам.
3. Раз в час сверяет показания с регламентом.
4. При превышении критического уровня получает уведомление (звук, всплывающее окно).
5. В зависимости от ситуации:
   - вызывает инженера;
   - сообщает технологу;
   - фиксирует событие в журнале.

### Сценарий 2: Инженер / технолог

**Цель:** своевременное реагирование на нештатные ситуации.

1. Получает уведомление о превышении параметра (аммиак, нитраты и т.п.).
2. Анализирует возможные причины:
   - поломка насоса;
   - сбой в очистных сооружениях;
   - истощение реагентов.
3. Принимает меры: запуск резервного оборудования, вызов ремонтной бригады.
4. При необходимости просматривает историю изменений параметра.

### Сценарий 3: Учёный (исследователь)

**Цель:** анализ данных для выявления закономерностей.

1. Удалённо подключается к центральному серверу через VPN.
2. Запрашивает данные за определённый период с нескольких объектов.
3. Строит графики, ищет корреляции между:
   - содержанием азотистых соединений;
   - темпом роста рыб;
   - составом кормов.
4. Использует результаты для научных публикаций или рекомендаций производству.

### Сценарий 4: Администратор системы

**Цель:** обеспечение бесперебойной работы и безопасности.

1. Управляет учётными записями пользователей.
2. Назначает роли и уровни доступа.
3. Контролирует работу каналов передачи данных.
4. Организует резервное копирование локальных данных.
5. Обновляет ПО и настройки системы.
