## 3. Пользовательский образ системы

### 3.1. Назначение системы
Система предназначена для **автоматизированного химического контроля водной среды** в технологических процессах, связанных с:
- энергетикой,
- пищевым производством,
- аквабиотехнологиями и рыбоводством,
- очистными сооружениями.

### 3.2. Ключевые характеристики (как видит пользователь)

| Характеристика | Описание |
|----------------|----------|
| **Измерение** | Система получает данные от 8-канального прибора АТОН-801, установленного в потоке воды (на входе и выходе). |
| **Визуализация** | На экране оператора отображаются графики текущих значений с цветовой индикацией (зелёный, жёлтый, красный). |
| **Оповещение** | При выходе за границы система отправляет уведомления оператору, инженерам, технологам (в интерфейсе, по почте, в Telegram, SMS). |
| **Хранение** | Данные хранятся локально на предприятии и передаются на удалённый сервер для научных исследований. |
| **Интеграция** | Система работает с уже установленным оборудованием, не требуя его замены. |
| **Безопасность** | Доступ к данным защищён, передача идёт по VPN, разграничены роли пользователей. |
| **Администрирование** | Система управляется администратором, который контролирует доступ и резервное копирование. |

### 3.3. Ожидания пользователей от системы
- **Оператор:** «Я должен видеть, всё ли в порядке, и получать сигнал, если что-то пошло не так».
- **Инженер:** «Я хочу получать уведомления о критических отклонениях и быстро понимать, в чём проблема».
- **Учёный:** «Мне нужны накопленные данные за длительный период для исследований и поиска закономерностей».
- **Администратор:** «Я должен управлять доступом и обеспечивать сохранность данных».
