## 2. Пользовательские сценарии использования

### 2.1. Сценарий: Оператор контролирует параметры

**Актор:** Оператор предприятия

**Цель:** Убедиться в нормальном состоянии водной среды

**Основной сценарий:**
1. Оператор открывает интерфейс системы на пульте.
2. Видит графики и текущие значения по всем датчикам.
3. В штатном режиме проверяет показания по регламенту (например, раз в час).
4. При выходе параметра в «жёлтую зону» получает умеренное оповещение.
5. При выходе в «красную зону» получает настойчивое оповещение.
6. В зависимости от ситуации вызывает инженера, технолога или фиксирует событие.

### 2.2. Сценарий: Инженер получает уведомление об аварийной ситуации

**Актор:** Инженер / главный технолог

**Цель:** Оперативно отреагировать на отклонение параметров

**Основной сценарий:**
1. Инженер получает уведомление в Telegram/email/SMS о превышении критического уровня (например, аммиака).
2. Открывает удалённый доступ к системе.
3. Анализирует динамику изменений и исторические данные.
4. Принимает решение:
   - вызвать ремонтную бригаду;
   - проверить работу насосов;
   - проверить подачу реагентов.
5. Фиксирует факт реагирования.

### 2.3. Сценарий: Учёный анализирует накопленные данные

**Актор:** Научный сотрудник (ихтиолог, гидрохимик)

**Цель:** Выявить корреляции между составом воды и биологическими показателями

**Основной сценарий:**
1. Учёный подключается к центральному серверу через VPN.
2. Запрашивает данные за определённый период по нескольким объектам.
3. Сравнивает показатели аммония, нитратов, нитритов с темпами роста рыбы.
4. Строит графики, выгружает данные для статистической обработки.
5. Использует результаты для научных отчётов или рекомендаций производству.

### 2.4. Сценарий: Администратор управляет доступом

**Актор:** Администратор системы

**Цель:** Обеспечить безопасность и разграничение доступа

**Основной сценарий:**
1. Администратор создаёт учётные записи новых пользователей.
2. Назначает роли: оператор, инженер, технолог, учёный.
3. Настраивает права доступа к данным и интерфейсам.
4. Контролирует активность пользователей.
5. Организует резервное копирование данных.
